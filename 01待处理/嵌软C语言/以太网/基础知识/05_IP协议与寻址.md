# IP协议与寻址

> [!NOTE]
> 本笔记为以太网基础知识体系中的网络层核心笔记，覆盖 IPv4 首部结构、IP 地址分类与子网掩码、单片机 IP 配置方式以及 MTU 与 IP 分片机制。

---

## 1. 核心概念

IP（Internet Protocol）是整个 TCP/IP 协议栈的**寻址中枢**——它为每台设备分配一个全局唯一的逻辑地址（IP 地址），使数据包能够跨越多个网络从源端路由到目的端。在嵌入式以太网中，IP 是所有上层协议（TCP/UDP/ICMP）的承载体。

---

## 2. 原理详解

### 2.1 IPv4 首部结构

IPv4 馴部固定部分为 **20 字节**，其结构如下：

```c
struct __attribute__((packed)) ip_hdr {
    uint8_t  vhl;         /* 版本(4bit)=4 + 首部长度(4bit)*4=20字节 */
    uint8_t  tos;         /* 服务类型（一般设0） */
    uint16_t len;         /* IP包总长度（首部+数据） */
    uint16_t id;          /* 标识符（分片重组时用） */
    uint16_t offset;      /* 标志+片偏移（DF=禁止分片, MF=还有后续片） */
    uint8_t  ttl;         /* 生存时间：每经一个路由器减1，到0则丢弃 */
    uint8_t  proto;       /* 协议号：6=TCP, 17=UDP, 1=ICMP */
    uint16_t chksum;      /* 首部校验和 */
    uint32_t src;         /* 源 IP 地址（网络字节序） */
    uint32_t dest;        /* 目的 IP 地址（网络字节序） */
};
```

**关键字段速查**：

| 字段 | 字节数 | 作用 | 嵌入式注意点 |
|---|---|---|---|
| **vhl** | 1 | 版本+首部长度 | vhl = 0x45（版本4，首部5*4=20字节） |
| **len** | 2 | IP 包总长度 | 需 ntohs 读取；最大 65535，实际受 MTU 限制 |
| **ttl** | 1 | 生存时间 | 嵌入式通常设 64，防止包在网络中无限循环 |
| **proto** | 1 | 上层协议号 | 用于分发：TCP(6)/UDP(17)/ICMP(1) |
| **src/dest** | 4 | 源/目的 IP | 网络字节序，需 ntohl 转换后比较 |

> [!TIP]
> 多字节字段（len、id、offset、chksum、src、dest）全部为**网络字节序**，C 代码中读取时必须用 **ntohs/ntohl** 转换。详见 [[大小端无缝转换机制#核心转换函数与宏|大小端无缝转换机制]]。

---

### 2.2 IP 地址分类与子网掩码

IPv4 地址为 32 位（4 字节），分为**网络号**和**主机号**两部分：

| 类别 | 首字节范围 | 网络号位数 | 主机号位数 | 适用场景 |
|---|---|---|---|---|
| A 类 | 1~126 | 8 | 24 | 大型网络 |
| B 类 | 128~191 | 16 | 16 | 中型网络 |
| C 类 | 192~223 | 24 | 8 | 小型网络（嵌入式常见） |

**CIDR 表示法**：如 `192.168.1.10/24` 表示前 24 位是网络号，后 8 位是主机号。子网掩码为 `255.255.255.0`。

嵌入式项目中几乎全部使用 C 类私有地址：
- **192.168.x.x/24**：最常用的局域网地址段
- **10.x.x.x/8**：大型私有网络
- **172.16~31.x.x/16**：中型私有网络

### 2.3 单片机 IP 配置：静态 vs DHCP

| 方式 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| **静态 IP** | 简单可靠，无额外协议栈开销 | 需手动规划，多设备易冲突 | 固定设备的工业场景 |
| **DHCP** | 自动获取，无需手动配置 | 需 DHCP 客户端代码，依赖 DHCP 服务器 | 动态接入的消费场景 |

```c
/* LwIP 静态 IP 配置示例 */
ip4_addr_t ipaddr, netmask, gw;
IP4_ADDR(&ipaddr,  192, 168, 1, 10);
IP4_ADDR(&netmask, 255, 255, 255, 0);
IP4_ADDR(&gw,      192, 168, 1, 1);
netif_add(&netif, &ipaddr, &netmask, &gw, NULL, ethernetif_init, tcpip_input);
```

> [!CAUTION]
> 静态 IP 配置时必须确保 **IP 不冲突**（同一子网内两台设备不能有相同 IP），否则 ARP 表会出现混乱，导致帧投递到错误设备。

---

### 2.4 MTU 与 IP 分片

**MTU**（Maximum Transmission Unit）是链路层一次能传输的最大数据量，以太网 MTU = **1500 字节**。如果 IP 包总长度超过 MTU，则需要进行**分片（Fragmentation）**：

- 发送端将大 IP 包拆成多个小片，每片独立加 IP 首部
- 接收端根据 **id 字段**（标识同一个原始包）和 **offset 字段**（片在原始包中的位置）重组

> [!WARNING]
> **嵌入式陷阱**：IP 分片大幅增加协议栈复杂度和内存消耗。LwIP 的 **PBUF_POOL** 机制可以处理分片重组，但配置不当会导致内存不足。**最佳实践**：在应用层控制发送数据大小不超过 MTU - 20(IP) - 20(TCP) = **1460 字节**，避免触发分片。

---

## 总结速查

- **IPv4 首部**固定 20 字节，关键字段：vhl(版本+长度)、len(包长度)、ttl(生存时间)、proto(协议号)、src/dest(IP地址)
- **IP 地址**32 位，嵌入式常用 C 类私有地址 `192.168.x.x/24`
- **静态 IP** 简单可靠适合工业场景，**DHCP** 自动获取适合消费场景
- **MTU = 1500 字节**，超过则 IP 分片；嵌入式最佳实践是应用层控制发送不超过 **1460 字节**
- 多字节字段全部网络字节序，需 **ntohs/ntohl** 转换
