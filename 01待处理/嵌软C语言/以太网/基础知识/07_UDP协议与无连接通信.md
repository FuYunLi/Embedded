# UDP协议与无连接通信

> [!NOTE]
> 本笔记为以太网基础知识体系中的传输层轻量协议笔记，覆盖 UDP 的极简首部结构、无连接特征、适用场景以及与 TCP 的核心对比。

---

## 1. 核心概念

UDP（User Datagram Protocol）是 TCP/IP 协议栈中**最轻量的传输层协议**——它不建立连接、不保证可靠、不做流量控制，唯一的职责是在 IP 包上加一个"端口"标签，让数据能投递到正确的应用。轻量的代价是：丢包不重传、乱序不重组、溢出不警告。但在嵌入式场景中，UDP 的简洁恰恰是优势——低开销、低延迟、适合广播。

---

## 2. 原理详解

### 2.1 UDP 首部结构

UDP 首部仅 **8 字节**，是所有传输层协议中最短的：

```c
struct __attribute__((packed)) udp_hdr {
    uint16_t src_port;   /* 源端口 */
    uint16_t dest_port;  /* 目的端口 */
    uint16_t len;        /* UDP 包总长度：首部(8) + 数据 */
    uint16_t chksum;     /* 校验和（可选，IPv4中可设为0表示不校验） */
};
```

与 TCP 的 20 字节首部相比，UDP 仅有 4 个字段，没有序列号、没有确认号、没有窗口大小。**简洁到极致**。

---

### 2.2 UDP 在嵌入式中的典型用法

LwIP RAW API 下 UDP 的使用比 TCP 更简单：

```c
/* 创建 UDP 控制块并绑定端口 */
struct udp_pcb *pcb = udp_new();
udp_bind(pcb, IP_ADDR_ANY, 8080);           /* 绑定本地端口 */

/* 注册接收回调 */
udp_recv(pcb, on_udp_recv, NULL);

/* 发送数据 */
struct pbuf *p = pbuf_alloc(PBUF_TRANSPORT, len, PBUF_RAM);
memcpy(p->payload, data, len);
udp_sendto(pcb, p, &dest_ip, dest_port);    /* 向目标发送 */
pbuf_free(p);

/* 接收回调 */
void on_udp_recv(void *arg, struct udp_pcb *pcb, struct pbuf *p,
                  const ip_addr_t *addr, uint16_t port) {
    /* 处理接收数据 */
    pbuf_free(p);
}
```

> [!TIP]
> UDP 没有连接概念，`udp_sendto` 直接指定目标 IP+端口即可发送，`udp_recv` 回调中自动获得来源 IP+端口。这使得 UDP 编程极其简洁——不需要 tcp_connect/tcp_accept 的握手流程。

---

### 2.3 TCP 与 UDP 对比

| 特性 | TCP | UDP |
|---|---|---|
| **连接性** | 面向连接（三次握手建立） | 无连接（直接发送） |
| **可靠性** | 可靠（确认+重传+有序） | 不可靠（丢包不通知、乱序不重组） |
| **首部开销** | 20 字节（最小） | 8 字节（固定） |
| **流量控制** | 滑动窗口 | 无 |
| **拥塞控制** | 有（慢启动等） | 无 |
| **传输效率** | 低（握手+确认开销） | 高（一发就完） |
| **适用场景** | HTTP、文件传输、MQTT | DNS、TDP、广播、实时数据上传 |

> [!IMPORTANT]
> **选择依据**：如果数据**不能丢**（如配置命令、文件传输）→ 用 TCP；如果数据**丢了也无所谓**或**时效性更重要**（如传感器周期上报、广播发现）→ 用 UDP。

---

### 2.4 UDP 的局限与应用层弥补

UDP 的三大局限：

1. **丢包**：网络拥塞或缓冲区满时数据包直接丢弃，发送方不知情
2. **乱序**：多个包可能以不同顺序到达，接收方需要自行排序
3. **无流量控制**：发送方速度过快会淹没接收方

**嵌入式中的弥补策略**：
- 应用层加**序列号**：检测丢包和乱序
- 应用层加**超时重传**：关键数据发送后等待 ACK，超时重发
- 应用层加**心跳包**：周期性确认双方存活
- 控制发送频率：避免突发流量淹没接收方

---

## 总结速查

- **UDP** 是最轻量的传输层协议，首部仅 8 字节（源端口+目的端口+长度+校验和）
- **无连接**：不需要握手，udp_sendto 直接指定目标即可发送
- **不可靠**：丢包不重传、乱序不重组、溢出不警告——但低延迟低开销
- **TCP vs UDP** 选择依据：数据不能丢→TCP；时效优先/丢了无所谓→UDP
- UDP 的局限可通过**应用层序列号+超时重传+心跳包**部分弥补
