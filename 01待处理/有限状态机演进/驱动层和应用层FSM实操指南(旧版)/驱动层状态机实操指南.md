# 驱动层状态机实操指南

> **适用场景**：硬件时序驱动（DHT11、I2C、SPI、UART协议、单总线设备等）
**核心关注**：精确时序、中断响应、非阻塞设计

---

## 核心思想：时序图 → 状态图

**驱动层状态机的本质**：把硬件时序图的每一个"等待点"转化为一个状态。

```Plaintext
[时序图中的每一段等待] = [状态机中的一个状态]
[时序图中的电平跳变]   = [状态机中的事件/触发条件]
```

---

## 版本一：逻辑构建（Switch\-Case 直接映射）

### 🎯 适用场景

- 状态数量 ≤ 5

- 时序简单，单一通信流程

- 快速原型验证

### 📖 原理讲解：DHT11 时序拆解

以 DHT11 为例，时序图如下：

```Plaintext
主机拉低18ms → 主机释放 → DHT11拉低80us → DHT11拉高80us → 40bit数据 → 结束
    ↓              ↓            ↓              ↓            ↓
 STATE_START   STATE_WAIT   STATE_RESP_L   STATE_RESP_H  STATE_DATA
```

**形象比喻**：就像**接力赛**，每个选手（状态）只负责跑自己的一棒（时序段），跑完交接棒（状态切换）给下一个。

### 📝 操作步骤清单

```Markdown
□ Step 1: 画时序图
  - 标注每一段的持续时间（us/ms）
  - 标注每一段的结束条件（电平跳变/超时）

□ Step 2: 定义状态枚举
  - 每一段时序 = 一个状态
  - 必须包含 IDLE（空闲）和 COMPLETE（完成）
  
□ Step 3: 定义返回值枚举
  - BUSY: 正在执行中
  - OK: 成功完成
  - ERROR_xxx: 各类错误（超时、校验失败等）

□ Step 4: 实现状态机主函数
  - switch(current_state) 结构
  - 每个 case 检查退出条件，满足则切换状态
  - 返回当前状态（BUSY/OK/ERROR）

□ Step 5: 实现中断处理（可选）
  - 用于捕获硬件事件（边沿跳变）
  - 中断只更新状态变量，不执行复杂逻辑

□ Step 6: 添加超时保护
  - 每个非 IDLE 状态都需要超时检测
  - 超时后回到 IDLE，返回错误码
```

### 💻 代码模板

```C
// ========== Step 2: 定义状态 ==========
typedef enum {
    DRV_STATE_IDLE,        // 空闲
    DRV_STATE_START,       // 起始信号
    DRV_STATE_WAIT_ACK,    // 等待响应
    DRV_STATE_DATA,        // 数据传输
    DRV_STATE_COMPLETE     // 完成
} drv_state_t;

// ========== Step 3: 定义返回值 ==========
typedef enum {
    DRV_STATUS_IDLE,       // 空闲
    DRV_STATUS_BUSY,       // 忙
    DRV_STATUS_OK,         // 成功
    DRV_STATUS_TIMEOUT,    // 超时
    DRV_STATUS_ERROR       // 错误
} drv_status_t;

// ========== Step 4: 状态机主函数 ==========
static drv_state_t g_state = DRV_STATE_IDLE;
static uint32_t g_timeout_cnt = 0;

drv_status_t drv_task(void)
{
    switch (g_state)
    {
    case DRV_STATE_IDLE:
        return DRV_STATUS_IDLE;
        
    case DRV_STATE_START:
        if (/* 起始信号完成条件 */) {
            g_timeout_cnt = 0;
            g_state = DRV_STATE_WAIT_ACK;
        }
        // Step 6: 超时检测
        if (++g_timeout_cnt > TIMEOUT_START) {
            g_state = DRV_STATE_IDLE;
            return DRV_STATUS_TIMEOUT;
        }
        return DRV_STATUS_BUSY;
        
    case DRV_STATE_WAIT_ACK:
        if (/* 收到响应 */) {
            g_state = DRV_STATE_DATA;
        }
        if (++g_timeout_cnt > TIMEOUT_ACK) {
            g_state = DRV_STATE_IDLE;
            return DRV_STATUS_TIMEOUT;
        }
        return DRV_STATUS_BUSY;
        
    case DRV_STATE_DATA:
        if (/* 数据传输完成 */) {
            g_state = DRV_STATE_COMPLETE;
        }
        return DRV_STATUS_BUSY;
        
    case DRV_STATE_COMPLETE:
        g_state = DRV_STATE_IDLE;
        return DRV_STATUS_OK;
        
    default:
        g_state = DRV_STATE_IDLE;
        return DRV_STATUS_ERROR;
    }
}
```

## 版本二：功能完善（封装 \+ ENTRY/EXIT）

### 🎯 适用场景

- 多实例驱动（如板上有 2 个 DHT11）

- 状态切换时需要资源管理（开关中断、使能外设）

- 需要可移植性

### 📖 原理讲解：结构体封装

**形象比喻**：版本一的代码像**独生子**，全局变量与代码绑定。版本二是**印刷模版**，一个模版可以"印"出多个独立运行的实例。

```C
// 版本一：全局变量，只能一个实例
static uint8_t g_state;
static uint8_t g_data[5];

// 版本二：结构体封装，支持多实例
typedef struct {
    uint8_t state;
    uint8_t data[5];
    GPIO_TypeDef *port;
    uint16_t pin;
} drv_inst_t;

drv_inst_t sensor1, sensor2;  // 两个独立实例
```

### 📖 原理讲解：ENTRY/EXIT 机制

进入新状态前执行 **ENTRY（初始化）**，离开旧状态前执行 **EXIT（清理）**。

```Plaintext
┌─────────────────┐
                    │  STATE_A EXIT   │ ← 离开 A 时执行
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  STATE_B ENTRY  │ ← 进入 B 时执行
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  STATE_B 主逻辑  │
                    └─────────────────┘
```

**实际用途**：

- ENTRY：使能外设时钟、配置 GPIO、启动定时器

- EXIT：关闭中断、释放资源、停止 DMA

### 📝 操作步骤清单

```Markdown
□ Step 1: 定义驱动实例结构体
  - 状态变量
  - 数据缓冲区
  - 硬件引脚配置
  - 超时计数器

□ Step 2: 修改所有函数签名
  - 第一个参数改为 drv_inst_t *self
  - 所有全局变量访问改为 self->xxx

□ Step 3: 实现状态切换函数（含 ENTRY/EXIT）
  - drv_tran(self, new_state)
  - 内部调用 old_state 的 EXIT
  - 再调用 new_state 的 ENTRY

□ Step 4: 定义 ENTRY/EXIT 处理表（可选）
  - 函数指针数组，按状态索引
  - 或直接在 drv_tran() 里用 switch 处理

□ Step 5: 初始化函数创建实例
  - drv_init(self, port, pin, ...)
  - 配置硬件，状态置为 IDLE
```

### 💻 代码模板

```C
// ========== Step 1: 驱动实例结构体 ==========
typedef struct drv_inst {
    drv_state_t state;
    uint8_t data[5];
    uint32_t timeout_cnt;
    // 硬件配置
    GPIO_TypeDef *port;
    uint16_t pin;
    TIM_TypeDef *tim;
} drv_inst_t;

// ========== Step 3: 状态切换函数 ==========
static void drv_tran(drv_inst_t *self, drv_state_t new_state)
{
    // EXIT: 离开旧状态的清理工作
    switch (self->state)
    {
    case DRV_STATE_DATA:
        // 例：关闭外部中断
        EXTI_Cmd(self->exti_line, DISABLE);
        break;
    default:
        break;
    }
    
    // 更新状态
    self->state = new_state;
    self->timeout_cnt = 0;
    
    // ENTRY: 进入新状态的初始化工作
    switch (new_state)
    {
    case DRV_STATE_START:
        // 例：配置 GPIO 为输出，拉低总线
        gpio_set_output(self->port, self->pin);
        gpio_write(self->port, self->pin, 0);
        break;
    case DRV_STATE_WAIT_ACK:
        // 例：配置 GPIO 为输入，使能外部中断
        gpio_set_input(self->port, self->pin);
        EXTI_Cmd(self->exti_line, ENABLE);
        break;
    default:
        break;
    }
}

// ========== Step 2: 状态机主函数（带 self 指针） ==========
drv_status_t drv_task(drv_inst_t *self)
{
    switch (self->state)
    {
    case DRV_STATE_IDLE:
        return DRV_STATUS_IDLE;
        
    case DRV_STATE_START:
        if (/* 条件满足 */) {
            drv_tran(self, DRV_STATE_WAIT_ACK);  // 使用封装的状态切换
        }
        if (++self->timeout_cnt > TIMEOUT) {
            drv_tran(self, DRV_STATE_IDLE);
            return DRV_STATUS_TIMEOUT;
        }
        return DRV_STATUS_BUSY;
        
    // ... 其他状态类似
    }
}

// ========== Step 5: 初始化函数 ==========
void drv_init(drv_inst_t *self, GPIO_TypeDef *port, uint16_t pin)
{
    self->port = port;
    self->pin = pin;
    self->state = DRV_STATE_IDLE;
    self->timeout_cnt = 0;
    memset(self->data, 0, sizeof(self->data));
    
    // 硬件初始化...
}
```

## 版本三：系统集成（中断解耦 \+ 异步通知）

### 🎯 适用场景

- 与 RTOS 集成

- 需要从中断中安全通知应用层

- 多驱动协同工作

### 📖 原理讲解：中断与主循环解耦

**问题**：中断中不能执行耗时操作，但驱动状态机可能需要复杂处理。

**解决方案**：中断只做两件事：

1. 捕获硬件事件（记录时间戳、电平状态）

2. 设置标志位或更新状态变量

主循环负责状态机逻辑处理。

```Plaintext
┌─────────────┐     标志位/队列     ┌─────────────┐
│   中断ISR   │ ─────────────────→ │  主循环Task  │
│  (10us内)   │                    │  (处理逻辑)  │
└─────────────┘                    └─────────────┘
```

**形象比喻**：中断像**前台接待**，只负责登记来客信息；主循环像**后台经理**，根据登记信息处理业务。

### 📖 原理讲解：回调通知机制

驱动层不应该直接调用应用层函数（打破分层），而是通过**回调函数**通知上层。

```C
// 驱动层定义回调类型
typedef void (*drv_callback_t)(drv_status_t status, void *data);

// 应用层注册回调
drv_set_callback(&sensor, on_sensor_complete);

// 驱动层在状态机完成时调用
if (self->callback) {
    self->callback(DRV_STATUS_OK, self->data);
}
```

### 📝 操作步骤清单

```Markdown
□ Step 1: 分离中断处理与状态机逻辑
  - 中断只更新：时间戳、电平状态、原始数据
  - 中断末尾设置标志位：self->irq_pending = 1

□ Step 2: 状态机检查中断标志
  - if (self->irq_pending) { 处理; self->irq_pending = 0; }
  - 或使用信号量/事件组（RTOS 环境）

□ Step 3: 定义回调函数类型
  - typedef void (*callback_t)(status, data);
  - 结构体中增加 callback 字段

□ Step 4: 提供回调注册接口
  - drv_set_callback(self, callback)

□ Step 5: 状态机完成时触发回调
  - 在 COMPLETE 状态或错误状态调用回调
  - 回调在主循环上下文执行，可安全操作

□ Step 6: （RTOS）使用消息队列替代标志位
  - 中断中 xQueueSendFromISR()
  - 任务中 xQueueReceive() 阻塞等待
```

### 💻 代码模板

```C
// ========== Step 3: 回调类型定义 ==========
typedef void (*drv_callback_t)(drv_status_t status, uint8_t *data, uint8_t len);

// ========== 增强的驱动实例结构体 ==========
typedef struct drv_inst {
    drv_state_t state;
    uint8_t data[5];
    uint32_t timeout_cnt;
    
    // 中断相关
    volatile uint8_t irq_pending;     // 中断标志
    volatile uint32_t irq_timestamp;  // 中断时间戳
    volatile uint8_t irq_level;       // 中断时电平
    
    // 回调相关
    drv_callback_t callback;
    void *user_data;
    
    // 硬件配置
    GPIO_TypeDef *port;
    uint16_t pin;
} drv_inst_t;

// ========== Step 4: 回调注册接口 ==========
void drv_set_callback(drv_inst_t *self, drv_callback_t cb, void *user_data)
{
    self->callback = cb;
    self->user_data = user_data;
}

// ========== Step 1: 中断处理（极简） ==========
void drv_irq_handler(drv_inst_t *self)
{
    // 只记录关键信息，不做逻辑判断
    self->irq_timestamp = timer_get_us();
    self->irq_level = gpio_read(self->port, self->pin);
    self->irq_pending = 1;  // 通知主循环
}

// ========== Step 2 & 5: 状态机主函数 ==========
drv_status_t drv_task(drv_inst_t *self)
{
    drv_status_t result;
    
    switch (self->state)
    {
    case DRV_STATE_DATA:
        // Step 2: 检查中断标志
        if (self->irq_pending) {
            self->irq_pending = 0;
            
            // 处理中断数据
            uint32_t pulse_width = self->irq_timestamp - self->last_timestamp;
            // ... 数据解析逻辑
            
            if (/* 数据接收完成 */) {
                drv_tran(self, DRV_STATE_COMPLETE);
            }
        }
        // 超时检测
        if (++self->timeout_cnt > TIMEOUT) {
            drv_tran(self, DRV_STATE_IDLE);
            result = DRV_STATUS_TIMEOUT;
            goto notify;
        }
        return DRV_STATUS_BUSY;
        
    case DRV_STATE_COMPLETE:
        drv_tran(self, DRV_STATE_IDLE);
        result = DRV_STATUS_OK;
        goto notify;
        
    // ... 其他状态
    }
    return DRV_STATUS_BUSY;

notify:
    // Step 5: 触发回调通知应用层
    if (self->callback) {
        self->callback(result, self->data, sizeof(self->data));
    }
    return result;
}

// ========== 应用层使用示例 ==========
void on_sensor_complete(drv_status_t status, uint8_t *data, uint8_t len)
{
    if (status == DRV_STATUS_OK) {
        printf("Temp: %d.%d\r\n", data[2], data[3]);
    } else {
        printf("Sensor error: %d\r\n", status);
    }
}

int main(void)
{
    drv_inst_t sensor;
    drv_init(&sensor, GPIOC, GPIO_Pin_3);
    drv_set_callback(&sensor, on_sensor_complete, NULL);
    
    drv_start(&sensor);
    
    while (1) {
        drv_task(&sensor);  // 非阻塞轮询
        // ... 其他任务
    }
}
```

## 总结：版本选型指南

|版本|状态数|实例数|RTOS|推荐场景|
|---|---|---|---|---|
|V1 逻辑构建|≤5|1|❌|快速原型、简单协议|
|V2 功能完善|5\-10|多个|❌|产品级驱动、多传感器|
|V3 系统集成|不限|多个|✅|复杂系统、RTOS集成|
