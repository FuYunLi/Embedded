# 应用层状态机实操指南

> **适用场景**：业务逻辑控制（设备模式切换、菜单系统、通信协议栈、工作流程控制）
**核心关注**：用户交互、模式管理、事件响应

---

## 核心思想：用户故事 → 状态图

**应用层状态机的本质**：把用户可感知的"阶段"转化为状态，把用户操作或系统事件转化为触发条件。

```Plaintext
[用户能感知的每个阶段]   = [状态机中的一个状态]
[按键/传感器/定时器事件] = [状态机中的事件]
[用户期望的响应]         = [状态切换 + 动作执行]
```

---

## 版本一：逻辑构建（Switch\-Case 快速实现）

### 🎯 适用场景

- 状态数量 ≤ 5

- 简单的模式切换（如开机/运行/待机）

- 快速验证产品逻辑

### 📖 原理讲解：智能台灯控制

**用户故事**：

- 用户短按开灯，再短按关灯

- 长按可以进入调光模式

- 双击紧急关闭

**形象比喻**：就像**交通信号灯**，每个颜色（状态）有明确的含义，按时间或事件切换。

```Plaintext
┌──────────┐  短按   ┌──────────┐
      │   关灯   │ ──────→ │   亮灯   │
      │ (STANDBY)│ ←────── │ (ON)     │
      └──────────┘  短按   └────┬─────┘
                               │长按
                               ▼
                         ┌──────────┐
                         │  调光模式 │
                         │ (DIMMING)│
                         └──────────┘
```

### 📝 操作步骤清单

```Markdown
□ Step 1: 列出用户可感知的所有"阶段"
  - 问自己：用户在这个阶段能做什么？看到什么？

□ Step 2: 定义状态枚举
  - 命名要有业务含义（ON/OFF/DIMMING 而非 STATE_1/2/3）
  
□ Step 3: 列出所有可能的事件
  - 按键事件：短按、长按、双击
  - 定时器事件：超时、周期触发
  - 传感器事件：阈值触发、数据就绪

□ Step 4: 画状态转移表
  - 表格形式：[当前状态] + [事件] → [下一状态] + [动作]

□ Step 5: 实现状态机处理函数
  - switch(state) + switch(event) 嵌套结构
  - 每个 case 执行动作并切换状态

□ Step 6: 在主循环中调用
  - 定时轮询或事件驱动调用
```

### 📊 状态转移表模板

|当前状态|事件|下一状态|动作|
|---|---|---|---|
|STANDBY|短按|ON|开启 LED|
|STANDBY|长按|STANDBY|无|
|ON|短按|STANDBY|关闭 LED|
|ON|长按|DIMMING|进入调光|
|DIMMING|短按|ON|退出调光|
|DIMMING|长按|STANDBY|关闭 LED|

### 💻 代码模板

```C
// ========== Step 2: 定义状态 ==========
typedef enum {
    APP_STATE_STANDBY,   // 待机
    APP_STATE_ON,        // 运行
    APP_STATE_DIMMING    // 调光模式
} app_state_t;

// ========== Step 3: 定义事件 ==========
typedef enum {
    APP_EVT_NONE,        // 无事件
    APP_EVT_KEY_SHORT,   // 短按
    APP_EVT_KEY_LONG,    // 长按
    APP_EVT_KEY_DOUBLE,  // 双击
    APP_EVT_TIMEOUT      // 超时
} app_event_t;

// ========== Step 5: 状态机处理函数 ==========
static app_state_t g_state = APP_STATE_STANDBY;

void app_fsm_handler(app_event_t event)
{
    // 外层：当前状态
    switch (g_state)
    {
    case APP_STATE_STANDBY:
        // 内层：发生的事件
        switch (event)
        {
        case APP_EVT_KEY_SHORT:
            led_on();
            g_state = APP_STATE_ON;
            break;
        case APP_EVT_KEY_DOUBLE:
            // 待机时双击无效
            break;
        default:
            break;
        }
        break;
        
    case APP_STATE_ON:
        switch (event)
        {
        case APP_EVT_KEY_SHORT:
            led_off();
            g_state = APP_STATE_STANDBY;
            break;
        case APP_EVT_KEY_LONG:
            dimmer_start();
            g_state = APP_STATE_DIMMING;
            break;
        case APP_EVT_KEY_DOUBLE:
            // 紧急关闭
            led_off();
            g_state = APP_STATE_STANDBY;
            break;
        default:
            break;
        }
        break;
        
    case APP_STATE_DIMMING:
        switch (event)
        {
        case APP_EVT_KEY_SHORT:
            dimmer_stop();
            g_state = APP_STATE_ON;
            break;
        case APP_EVT_KEY_LONG:
            dimmer_stop();
            led_off();
            g_state = APP_STATE_STANDBY;
            break;
        default:
            break;
        }
        break;
    }
}

// ========== Step 6: 主循环调用 ==========
int main(void)
{
    hardware_init();
    
    while (1) {
        app_event_t evt = key_scan();  // 获取按键事件
        if (evt != APP_EVT_NONE) {
            app_fsm_handler(evt);
        }
    }
}
```

## 版本二：功能完善（查表法 \+ ENTRY/EXIT）

### 🎯 适用场景

- 状态数量 5\-15

- 需要状态进入/退出时执行初始化/清理

- 多人协作开发，需要代码解耦

### 📖 原理讲解：函数指针查表

**版本一的问题**：状态越多，switch\-case 越长，改一个状态容易误伤其他状态。

**解决方案**：把每个状态的处理逻辑封装成独立函数，用数组索引直接调用。

```C
// 版本一：O(n) 查找
switch(state) {
    case STATE_A: handle_A(); break;
    case STATE_B: handle_B(); break;
    case STATE_C: handle_C(); break;
    // ... 状态越多，分支越多
}

// 版本二：O(1) 查表
state_handler_t handlers[] = {handle_A, handle_B, handle_C};
handlers[state](event);  // 一行搞定
```

**形象比喻**：版本一像**挨个敲门问"你是张三吗？"**，版本二像**直接查通讯录打电话**。

### 📖 原理讲解：ENTRY/EXIT 在应用层的价值

|应用场景|ENTRY 动作|EXIT 动作|
|---|---|---|
|进入菜单模式|显示菜单界面|隐藏菜单|
|进入低功耗模式|关闭外设时钟、设置唤醒源|恢复时钟、刷新显示|
|进入配对模式|开启蓝牙广播、闪烁LED|停止广播|
|进入报警状态|启动蜂鸣器、发送通知|关闭蜂鸣器|

### 📝 操作步骤清单

```Markdown
□ Step 1: 定义状态处理函数类型
  - typedef void (*state_handler_t)(app_event_t event);

□ Step 2: 为每个状态编写独立的处理函数
  - static void handle_standby(app_event_t event);
  - static void handle_on(app_event_t event);
  - ...

□ Step 3: 建立函数指针数组（状态表）
  - 数组下标 = 状态枚举值
  - 数组元素 = 对应的处理函数

□ Step 4: 实现 ENTRY/EXIT 函数（可选）
  - 每个状态可以有自己的 entry() 和 exit()
  - 或用 switch-case 统一处理

□ Step 5: 实现状态切换函数
  - 调用旧状态的 exit()
  - 更新状态变量
  - 调用新状态的 entry()

□ Step 6: 简化调度器
  - handlers[g_state](event);  // 一行代替整个 switch
```

### 💻 代码模板

```C
// ========== Step 1: 定义函数指针类型 ==========
typedef void (*state_handler_t)(app_event_t event);
typedef void (*state_action_t)(void);

// ========== Step 2: 状态处理函数 ==========
static void handle_standby(app_event_t event)
{
    switch (event)
    {
    case APP_EVT_KEY_SHORT:
        app_tran(APP_STATE_ON);
        break;
    default:
        break;
    }
}

static void handle_on(app_event_t event)
{
    switch (event)
    {
    case APP_EVT_KEY_SHORT:
        app_tran(APP_STATE_STANDBY);
        break;
    case APP_EVT_KEY_LONG:
        app_tran(APP_STATE_DIMMING);
        break;
    default:
        break;
    }
}

static void handle_dimming(app_event_t event)
{
    switch (event)
    {
    case APP_EVT_KEY_SHORT:
        app_tran(APP_STATE_ON);
        break;
    case APP_EVT_TIMEOUT:
        brightness_step();  // 周期性调节亮度
        break;
    default:
        break;
    }
}

// ========== Step 3: 状态处理表 ==========
static const state_handler_t g_handlers[] = {
    [APP_STATE_STANDBY] = handle_standby,
    [APP_STATE_ON]      = handle_on,
    [APP_STATE_DIMMING] = handle_dimming,
};

// ========== Step 4: ENTRY/EXIT 函数 ==========
static void entry_standby(void) { led_off(); oled_show("待机"); }
static void exit_standby(void)  { /* 无需清理 */ }

static void entry_on(void)      { led_on(); oled_show("运行中"); }
static void exit_on(void)       { /* 无需清理 */ }

static void entry_dimming(void) { timer_start(100); oled_show("调光模式"); }
static void exit_dimming(void)  { timer_stop(); }

static const state_action_t g_entry[] = {
    [APP_STATE_STANDBY] = entry_standby,
    [APP_STATE_ON]      = entry_on,
    [APP_STATE_DIMMING] = entry_dimming,
};

static const state_action_t g_exit[] = {
    [APP_STATE_STANDBY] = exit_standby,
    [APP_STATE_ON]      = exit_on,
    [APP_STATE_DIMMING] = exit_dimming,
};

// ========== Step 5: 状态切换函数 ==========
static app_state_t g_state = APP_STATE_STANDBY;

static void app_tran(app_state_t new_state)
{
    if (new_state == g_state) return;  // 防止自循环
    
    // 执行旧状态的 EXIT
    if (g_exit[g_state]) {
        g_exit[g_state]();
    }
    
    // 切换状态
    app_state_t old_state = g_state;
    g_state = new_state;
    
    // 执行新状态的 ENTRY
    if (g_entry[g_state]) {
        g_entry[g_state]();
    }
    
    printf("状态切换: %d -> %d\r\n", old_state, new_state);
}

// ========== Step 6: 简化调度器 ==========
void app_fsm_dispatch(app_event_t event)
{
    if (g_handlers[g_state]) {
        g_handlers[g_state](event);
    }
}

// ========== 主循环 ==========
int main(void)
{
    hardware_init();
    g_entry[g_state]();  // 初始状态的 ENTRY
    
    while (1) {
        app_event_t evt = event_poll();
        if (evt != APP_EVT_NONE) {
            app_fsm_dispatch(evt);
        }
    }
}
```

## 版本三：系统集成（消息队列 \+ 事件参数）

### 🎯 适用场景

- 与 RTOS 集成

- 事件来源多样（按键、串口、传感器、定时器）

- 事件需要携带数据（如串口收到的字节）

- 多状态机协同工作

### 📖 原理讲解：消息队列解耦

**问题**：事件产生者（按键中断、UART 中断）和消费者（状态机）耦合太紧。

**解决方案**：引入消息队列作为缓冲区。

```Plaintext
┌──────────┐     ┌─────────────┐     ┌──────────┐
│ 按键中断  │────→│             │────→│          │
├──────────┤     │  消息队列   │     │  状态机   │
│ UART中断  │────→│  (Buffer)   │────→│  (消费)  │
├──────────┤     │             │     │          │
│ 定时器    │────→│             │     │          │
└──────────┘     └─────────────┘     └──────────┘
   生产者            中间件              消费者
```

**优势**：

1. **削峰填谷**：短时间大量事件不会丢失

2. **解耦时序**：中断快速返回，状态机按自己节奏处理

3. **统一接口**：所有事件走同一个入口

### 📖 原理讲解：事件携带参数

**问题**：事件不仅需要类型，还需要数据。

```C
// 版本二：事件只有类型
typedef enum { EVT_UART_RX, EVT_KEY, ... } event_t;

// 版本三：事件携带载荷
typedef struct {
    event_t type;           // 事件类型
    union {
        uint8_t  key_code;  // 按键码
        uint8_t  uart_byte; // 串口数据
        int16_t  sensor_val;// 传感器值
    } data;
} event_msg_t;
```

**形象比喻**：版本二是**发信号弹**（只知道有事），版本三是**发快递包裹**（知道具体内容）。

### 📝 操作步骤清单

```Markdown
□ Step 1: 定义事件消息结构体
  - 事件类型 + 数据载荷（union 节省空间）

□ Step 2: 实现消息队列（或使用 RTOS 队列）
  - 裸机：环形缓冲区 + 读写指针
  - RTOS：xQueueCreate()

□ Step 3: 事件生产者发送消息
  - 中断中：queue_send() 或 xQueueSendFromISR()
  - 不要在中断中处理逻辑！

□ Step 4: 修改状态处理函数签名
  - void handle_xxx(event_msg_t *msg);
  - 从 msg->data 中提取具体参数

□ Step 5: 主循环消费消息
  - while (queue_receive(&msg)) { dispatch(&msg); }

□ Step 6: （可选）多状态机协同
  - 每个状态机有自己的队列
  - 或共享队列 + 消息路由
```

### 💻 代码模板

```C
// ========== Step 1: 事件消息结构体 ==========
typedef struct {
    app_event_t type;       // 事件类型
    uint32_t timestamp;     // 时间戳（可选，用于调试）
    union {
        struct {
            uint8_t code;   // 按键码
            uint8_t action; // 0:按下 1:释放 2:长按
        } key;
        struct {
            uint8_t *data;
            uint16_t len;
        } uart;
        struct {
            int16_t temp;
            int16_t humi;
        } sensor;
    } payload;
} event_msg_t;

// ========== Step 2: 简易环形队列 ==========
#define QUEUE_SIZE 16

typedef struct {
    event_msg_t buffer[QUEUE_SIZE];
    volatile uint8_t head;
    volatile uint8_t tail;
} event_queue_t;

static event_queue_t g_queue = {0};

// 发送（中断安全）
uint8_t queue_send(event_msg_t *msg)
{
    uint8_t next = (g_queue.head + 1) % QUEUE_SIZE;
    if (next == g_queue.tail) {
        return 0;  // 队列满
    }
    g_queue.buffer[g_queue.head] = *msg;
    g_queue.head = next;
    return 1;
}

// 接收

uint8_t queue_receive(event_msg_t *msg)
{
    if (g_queue.head == g_queue.tail) {
        return 0;  // 队列空
    }
    *msg = g_queue.buffer[g_queue.tail];
    g_queue.tail = (g_queue.tail + 1) % QUEUE_SIZE;
    return 1;
}

// ========== Step 3: 事件生产者（中断） ==========
void KEY_IRQHandler(void)
{
    event_msg_t msg = {
        .type = APP_EVT_KEY,
        .timestamp = systick_get_ms(),
        .payload.key.code = get_key_code(),
        .payload.key.action = get_key_action()
    };
    queue_send(&msg);
}

void UART_IRQHandler(void)
{
    static uint8_t rx_buf[64];
    static uint8_t rx_len = 0;
    
    rx_buf[rx_len++] = UART_ReceiveData(UART1);
    
    if (/* 帧结束条件 */) {
        event_msg_t msg = {
            .type = APP_EVT_UART_RX,
            .payload.uart.data = rx_buf,
            .payload.uart.len = rx_len
        };
        queue_send(&msg);
        rx_len = 0;
    }
}

// ========== Step 4: 状态处理函数（接收消息指针） ==========
static void handle_standby(event_msg_t *msg)
{
    switch (msg->type)
    {
    case APP_EVT_KEY:
        if (msg->payload.key.action == KEY_ACTION_SHORT) {
            app_tran(APP_STATE_ON);
        }
        break;
        
    case APP_EVT_UART_RX:
        // 待机时收到串口命令，可以唤醒
        if (msg->payload.uart.len > 0 && 
            msg->payload.uart.data[0] == CMD_WAKEUP) {
            app_tran(APP_STATE_ON);
        }
        break;
        
    default:
        break;
    }
}

static void handle_on(event_msg_t *msg)
{
    switch (msg->type)
    {
    case APP_EVT_KEY:
        if (msg->payload.key.action == KEY_ACTION_SHORT) {
            app_tran(APP_STATE_STANDBY);
        } else if (msg->payload.key.action == KEY_ACTION_LONG) {
            app_tran(APP_STATE_DIMMING);
        }
        break;
        
    case APP_EVT_SENSOR:
        // 传感器数据到达时更新显示
        oled_printf("T:%d H:%d", 
            msg->payload.sensor.temp, 
            msg->payload.sensor.humi);
        break;
        
    default:
        break;
    }
}

// ========== 调度器 ==========
typedef void (*msg_handler_t)(event_msg_t *msg);

static const msg_handler_t g_handlers[] = {
    [APP_STATE_STANDBY] = handle_standby,
    [APP_STATE_ON]      = handle_on,
    [APP_STATE_DIMMING] = handle_dimming,
};

void app_fsm_dispatch(event_msg_t *msg)
{
    if (g_handlers[g_state]) {
        g_handlers[g_state](msg);
    }
}

// ========== Step 5: 主循环消费消息 ==========
int main(void)
{
    hardware_init();
    g_entry[g_state]();
    
    event_msg_t msg;
    
    while (1) {
        // 消费所有待处理消息
        while (queue_receive(&msg)) {
            app_fsm_dispatch(&msg);
        }
        
        // 其他后台任务
        idle_task();
    }
}
```

### 💡 RTOS 版本简化

// FreeRTOS 版本
QueueHandle_t g_event_queue;

void app_task(void *param)
{
    event_msg_t msg;
    
    while (1) {
        // 阻塞等待，不浪费 CPU
        if (xQueueReceive(g_event_queue, &msg, portMAX_DELAY)) {
            app_fsm_dispatch(&msg);
        }
    }
}

void KEY_IRQHandler(void)
{
    event_msg_t msg = { .type = APP_EVT_KEY, ... };
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(g_event_queue, &msg, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

## 总结：版本选型指南

|版本|状态数|事件来源|RTOS|推荐场景|
|---|---|---|---|---|
|V1 逻辑构建|≤5|单一（按键）|❌|快速原型、简单产品|
|V2 功能完善|5\-15|少量（2\-3种）|❌|成熟产品、多模式设备|
|V3 系统集成|不限|多样（中断/网络/传感器）|✅|物联网设备、复杂系统|

## 驱动层 vs 应用层 对比速查表

|维度|驱动层状态机|应用层状态机|
|---|---|---|
|**状态粒度**|细（us 级时序节点）|粗（用户可感知阶段）|
|**事件来源**|硬件中断|按键/定时器/外部消息|
|**时间约束**|硬实时（必须响应）|软实时（可以缓冲）|
|**典型状态数**|5\-10|3\-7|
|**ENTRY/EXIT 用途**|外设使能/关闭、GPIO 配置|UI 刷新、资源申请/释放|
|**多实例需求**|常见（多传感器）|少见|
|**调试关注点**|时序波形、超时|状态流转日志、用户行为|
